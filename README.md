## 📊 Interpretando Google Trends

### O Que Significam os Números?

```
Interesse: 0-100 (relatividade)
├─ 100 = Pico de interesse
├─ 50 = Metade do pico
└─ 0 = Sem volume de busca

Tendência:
├─ ⬆️ +50% = Crescendo MUITO
├─ ⬆️ +20% = Crescendo
├─ ⬇️ -10% = Caindo
└─ ⬇️ -50% = Caindo MUITO
```

### Quando Usar Cada Nível

```
⬆️ +70%+ = FOMO + Urgência 🚀
         (Todos procurando!)

⬆️ +30-70% = Urgência + Curiosidade 📈
           (Trending forte)

⬆️ +10-30% = Desconto + Social Proof 💰
           (Bom interesse)

⬇️ Negativo = Exclusividade + Storytelling 👑
           (Público selecionado)
```

## ⚠️ Cuidados & Limitações

### ✅ Funciona Bem
```
✅ Produtos em alta demanda
✅ Nichos específicos (moda, beleza)
✅ Tendências sazonais (Natal, verão)
✅ Produtos novos em alta
```

### ⚠️ Cuidado Com
```
⚠️ Muito genérico ("produto", "loja")
⚠️ Misspellings ("bolsa de ombr" vs "bolsa de ombro")
⚠️ Produtos muito obscuros (sem volume)
⚠️ Regional (trending muda por região)
```

### 🚫 Não Funciona
```
❌ Palavras vazias ("a", "o", "de")
❌ Buscas muito gerais
❌ Termos não indexados (nomes próprios)
❌ Volumes muito baixos
```

## 🎯 O Que Aconteceu

```
Erro ao buscar 'case celular': The request failed: 
Google returned a response with code 400
```

**Motivo:** Google Trends bloqueia requisições muito rápidas (throttling)

---

## ✅ SOLUÇÃO APLICADA

Atualizei o código com **3 melhorias**:

### 1️⃣ Delay Maior Entre Requisições
```python
# Antes: time.sleep(1)
# Agora: time.sleep(2)  ← 2 segundos entre cada requisição
```

### 2️⃣ Palavras-Chave Mais Genéricas
```python
# Antes: ["bolsa de ombro", "sérum facial", "carregador rápido"]
# Agora: ["bolsa", "sérum", "carregador"]  ← Mais simples, menos erro 400
```

### 3️⃣ Tratamento Silencioso de Erros
```python
# Antes: Mostrava erro pra cada falha
# Agora: Continua silenciosamente, usa cache se tiver
```
